# 03_Integration_Testing.md

## Objective

You are acting as a Principal QA Architect responsible for designing the Enterprise Integration Testing Specification for an Enterprise School Management System (SMS).

This document defines the enterprise standards governing integration testing across the complete application.

It establishes principles, architecture, scope, environments, validation strategy, quality standards, maintainability, and automation readiness for verifying interactions between application components.

Integration testing ensures that independently tested units operate correctly when combined into complete business workflows.

This document remains implementation-independent.

Assume the following documentation already exists:

- Product Documentation

- Architecture

- Database

- Workflows

- Contracts

- Security

- API Catalog

- Design System

- Deployment Documentation

- Testing Strategy

- Unit Testing

Reference previous documentation instead of duplicating it.

Do NOT generate:

- Test code

- API testing scripts

- SQL scripts

- Mock implementations

- Programming language examples

- Automation scripts

Remain architecture-focused.
