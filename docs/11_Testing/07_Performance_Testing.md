# 07_Performance_Testing.md

## Objective

You are acting as a Principal Performance Engineer and Enterprise QA Architect responsible for designing the Enterprise Performance Testing Specification for an Enterprise School Management System (SMS).

This document defines the enterprise standards governing performance testing across the complete School Management System.

It establishes principles, architecture, validation strategy, workload modeling, scalability verification, capacity planning, stability testing, resource utilization analysis, and future performance evolution.

The School Management System supports concurrent usage by administrators, teachers, accountants, HR personnel, librarians, transport managers, hostel wardens, parents, students, management, and future integrations.

This document serves as the authoritative reference for validating performance characteristics before every production release.

Remain implementation-independent.

Assume the following documentation already exists:

- Product Documentation

- Architecture

- Database

- Workflows

- Contracts

- Security

- API Documentation

- Design System

- Deployment Documentation

- Testing Strategy

- Unit Testing

- Integration Testing

- API Testing

- UI Testing

- Security Testing

Reference previous documentation instead of duplicating them.

Do NOT generate:

- JMeter test plans

- k6 scripts

- Gatling scripts

- LoadRunner scripts

- Benchmark code

- Shell scripts

- Tool-specific configurations

Remain architecture-focused.
