# 02_Infrastructure.md

## Objective

You are acting as a Principal Enterprise Infrastructure Architect responsible for designing the Infrastructure Architecture Specification for an Enterprise School Management System (SMS).

This document defines the complete enterprise infrastructure architecture required to deploy, secure, operate, monitor, and scale the School Management System.

Unlike the Server Architecture document, this specification focuses on the entire infrastructure ecosystem including networking, storage, security boundaries, environments, operational services, infrastructure dependencies, resilience, and future cloud readiness.

This document remains implementation-independent.

Assume the following documentation already exists:

- Product Documentation
- Architecture
- Database
- Workflows
- Contracts
- Security
- API Catalog
- Design System
- Server Architecture

Reference previous documentation instead of duplicating it.

Do NOT generate:

- Terraform

- Kubernetes YAML

- Docker Compose

- Shell Scripts

- CloudFormation

- Azure ARM Templates

- Server Commands

- Vendor-specific implementations

# Writing Style

Maintain the tone of a professional Enterprise Infrastructure Architecture Specification.

Avoid implementation details.

Avoid vendor-specific guidance.

Avoid cloud-provider-specific recommendations.

Avoid infrastructure-as-code examples.

Focus entirely on enterprise infrastructure architecture, security, resilience, scalability, observability, maintainability, operational excellence, and future evolution.

This document should become the definitive infrastructure blueprint for deploying the School Management System across on-premises, cloud, hybrid, and future SaaS environments.
