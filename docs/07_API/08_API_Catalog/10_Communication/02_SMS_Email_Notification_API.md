# 02_SMS_Email_Notification_API.md

## Objective

You are acting as a Principal Enterprise API Architect responsible for designing the complete API Catalog for the Enterprise Notification Delivery domain of an Enterprise School Management System (SMS).

This document is the authoritative specification governing every business API related to notification delivery, SMS, Email, Push Notifications, WhatsApp readiness, in-app messaging, channel routing, templates, delivery tracking, retries, scheduling, preferences, communication governance, and institutional notification administration.

The Notification Delivery domain is responsible for delivering communication generated by business modules including Announcements, Admissions, Attendance, Examination, Fees, Timetable, Homework, LMS, Parent Portal, HRMS, Transport, Hostel, Library, Inventory, Events, Alumni, and other future modules.

This document manages delivery only.

Business modules remain responsible for deciding WHAT to communicate.

This domain decides HOW communication is delivered.

This document must remain implementation-independent.

The architecture must support both:

- Server-rendered PHP MVC
- Headless REST API

through a common business service layer.

Do NOT generate:

- PHP
- SQL
- JSON payloads
- OpenAPI specifications
- Endpoint URLs
- Infrastructure
- SMS gateway implementation
- Email provider implementation
- Push provider implementation
- WhatsApp provider implementation

Assume all previously generated architecture, contracts, validation rules, workflows, security documentation, API standards, Announcement APIs, Student APIs, LMS APIs, and Reporting APIs already exist.

Reference previous documentation instead of duplicating it.

# Enterprise Notification Principles

Notification delivery is channel independent.

Business modules publish notification requests.

Notification Delivery selects delivery channels.

Templates remain reusable.

Deliveries remain auditable.

Retries never duplicate successfully delivered notifications.

Delivery status remains immutable.

Every delivery attempt is recorded.
