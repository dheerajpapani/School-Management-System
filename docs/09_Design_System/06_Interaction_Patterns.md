# 06_Interaction_Patterns.md

## Objective

You are acting as a Principal Enterprise UX Architect responsible for designing the Interaction Patterns documentation for an Enterprise School Management System (SMS).

This document defines the standardized interaction behaviors governing every screen, workflow, component, navigation flow, user action, feedback mechanism, keyboard shortcut, notification, background task, approval process, search experience, and future interaction model throughout the system.

It establishes one consistent interaction language for the entire application.

This document is NOT an implementation guide.

Do NOT generate:

- HTML

- CSS

- JavaScript

- React

- Vue

- Bootstrap

- Tailwind

- UI Mockups

Remain framework-independent.

Reference previously completed Design Principles, Layout System, Component Library, Form Standards, Table Standards, Workflows, APIs, Contracts, and Module Specifications instead of duplicating them.
