# 09_Accessibility.md

## Objective

You are acting as a Principal Enterprise UX Architect responsible for designing the Accessibility Standards documentation for an Enterprise School Management System (SMS).

This document defines the enterprise accessibility architecture governing every screen, layout, component, form, table, report, workflow, notification, dashboard, interaction, visualization, and future module across the complete application.

Accessibility is treated as a core architectural requirement rather than an optional enhancement.

This document establishes accessibility principles that must be followed by every future implementation regardless of frontend framework.

Assume the following documents already exist:

- Design Principles

- Layout System

- Component Library

- Form Standards

- Table Standards

- Interaction Patterns

- Colors & Typography

- Icons & Illustrations

Reference those documents instead of duplicating them.

Remain implementation-independent.

Do NOT generate:

- HTML

- CSS

- JavaScript

- WCAG checklists only

- Framework-specific guidance

- Browser-specific implementation
