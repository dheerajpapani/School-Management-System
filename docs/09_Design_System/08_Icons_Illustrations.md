# 08_Icons_Illustrations.md

## Objective

You are acting as a Principal Enterprise UX Architect responsible for designing the Icons & Illustrations documentation for an Enterprise School Management System (SMS).

This document defines the enterprise standards governing iconography, illustrations, visual symbols, imagery usage, empty states, onboarding graphics, workflow graphics, dashboard visuals, status indicators, educational imagery, accessibility, and future visual assets across the complete application.

This document establishes a consistent visual language for icons and illustrations while remaining implementation-independent.

It is NOT an icon library.

It is NOT a graphic asset repository.

It defines usage standards.

Assume all previous Design System documentation already exists including:

- Design Principles
- Layout System
- Component Library
- Form Standards
- Table Standards
- Interaction Patterns
- Colors & Typography

Reference previous documentation instead of duplicating it.

Remain framework-independent.

Do NOT generate:

- SVG

- PNG

- Icon Sets

- HTML

- CSS

- React Components

- Figma Assets

- Design Tokens

- Specific Icon Libraries

# Enterprise Visual Symbol Philosophy

Icons should improve recognition.

Icons must never replace important text.

Illustrations should communicate purpose.

Visual assets must support productivity rather than decoration.

Consistency takes priority over creativity.

Accessibility must always be maintained.

Visual language must remain scalable for future modules.
