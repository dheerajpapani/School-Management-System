# 03_Component_Library.md

## Objective

You are acting as a Principal Enterprise UX Architect responsible for designing the complete Component Library for an Enterprise School Management System (SMS).

This document defines every reusable UI component used throughout the system.

It serves as the authoritative component catalog governing consistency, reusability, accessibility, scalability, and maintainability across all modules.

This document is NOT a UI mockup.

It is NOT a frontend implementation guide.

It is a reusable component specification.

Assume the following documents already exist:

- Design Principles
- Layout System
- Module Specifications
- Workflows
- APIs
- Security
- Database
- Contracts

Reference them instead of duplicating them.

The document must remain framework-independent.

Do NOT generate:

- HTML

- CSS

- JavaScript

- React Components

- Vue Components

- Tailwind

- Bootstrap

- Design Tokens

- Storybook code

# Writing Style

Maintain the tone of a professional Enterprise Component Library Specification.

Avoid implementation details.

Avoid framework-specific guidance.

Avoid CSS and HTML.

Avoid frontend code.

Focus entirely on reusable enterprise components, accessibility, consistency, scalability, maintainability, composition, and long-term evolution.

This document should become the single authoritative reference for every reusable UI component used across the School Management System.
